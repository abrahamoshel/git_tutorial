Git Part II: When Things Go Wrong

~/tkml/projects/

Warning!

  • Time traveling and undoing things can alter the future! (leave your database in an inconsistent state)
  • This may result in having to re-run db migrations.
~/tkml/projects/

git revert

  • Remove / Undo Committed Code
  • Creates an inverse COMMIT of a previous commit
  • This makes the commit in whatever branch your in!
  • If your testing things out you should create a branch first e.g git checkout -b myFixBranch
  • Example git revert e1c5a54
~/tkml/projects/

git log

  • A useful tool for dealing with issues
  • (by default) shows the commit history from the tip of your current branch.
  • Useful for seeing when things happened.
  • Gives you sha's for doing other operations, such as revert.
~/tkml/projects/

git reflog

  • Shows your whole history of actions such as merges, checkouts, commits etc.
  • This is your safety net. You can use this to give you a better understanding of what has happened and how to get back to stability.
  • Useful if you get lost in time travel. Your map back home.
  • You have to try really hard to loose data in git. Things may seem like they are lost but reflog will help you find the data again.
~/tkml/projects/

git reset

  • Changes HEAD (the tip of your current branch) to the sha you indicate.
  • Useful for time travel
  • --hard option will change all files in your directory to that point in time blowing away all modified files
  • Not specifying --hard will move your HEAD pointer but will keep all files and indicate them as modified.
~/tkml/projects/

I made a commit that I didn't push that I don't want... What to do?

  • git reset --hard sha_here
  • Make sure to stash or branch off any un-committed files as they will be blown away!
~/tkml/projects/

I'm on detached head... What the heck is that?

  • If you do git branch you will see * (no branch)
  • You are no longer working on a branch
  • You got here by checking out a particular commit.
  • Commits made here are in never never land... (sort of)
  • If you have not made any changes or commits, you can get home by git checkout the branch you want to be on.
